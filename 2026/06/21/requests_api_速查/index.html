requests API 速查

本文按功能分类整理 Python 第三方库 requests 的常用 API，包括 GET/POST 请求、参数、请求头、JSON、Cookie、Session、文件上传下载、异常处理和代理设置。

requests 是 Python 爬虫和 API 请求里最常用的 HTTP 客户端库。相比标准库 urllib，它的写法更直观，尤其适合处理参数、JSON、headers、cookies 和登录会话。

1. GET 请求

requests.get(url, params=None, **kwargs)

发送 GET 请求。

参数：

  • url：请求地址
  • params：URL 查询参数，字典或键值对列表
  • **kwargs：通用请求参数，如 headers、cookies、timeout、proxies

示例：

params 会自动进行 URL 编码，不需要手动 urlencode()。

2. POST 表单请求

requests.post(url, data=None, json=None, **kwargs)

发送 POST 请求。

参数：

  • url：请求地址
  • data：表单数据，通常是字典、元组列表或 bytes
  • json：JSON 数据，传入字典后自动序列化，并设置 JSON 请求头
  • **kwargs：通用请求参数

表单 POST：

3. POST JSON 请求

使用 json= 参数发送 JSON。

不要把字典手动 json.dumps() 后再传给 json=。如果已经转成字符串，应传给 data= 并自己设置 headers。

4. 通用请求方法

requests.request(method, url, **kwargs)

底层通用请求入口，get()、post()、put() 等都是基于它。

参数：

  • method：请求方法，如 "GET"、"POST"、"PUT"、"DELETE"
  • url：请求地址
  • **kwargs：通用请求参数

常用快捷方法：

5. 设置请求头

headers

请求头字典。

常见 headers：

  • User-Agent：浏览器或客户端标识
  • Referer：来源页面
  • Accept：希望接收的数据类型
  • Content-Type：请求体类型
  • Authorization：鉴权信息

6. 读取响应内容

resp.text

按文本读取响应内容。

resp.content

按 bytes 读取响应内容，适合图片、字体、压缩包。

resp.json(**kwargs)

把响应体解析成 JSON。

参数：

  • **kwargs：传给 json.loads() 的参数

resp.encoding

响应文本编码。可以手动设置。

resp.apparent_encoding

requests 推测出来的编码。

7. 响应状态和响应头

resp.status_code

HTTP 状态码。

resp.headers

响应头，类似字典。

resp.url

最终请求 URL。

resp.history

重定向历史。

resp.raise_for_status()

如果状态码是 4xx 或 5xx，抛出 HTTPError。

8. 超时设置

timeout

设置请求超时时间。

参数形式：

  • timeout=10：连接和读取总超时都按 10 秒处理
  • timeout=(3, 15)：连接超时 3 秒，读取超时 15 秒

爬虫代码里建议总是设置 timeout，避免网络卡住。

9. Cookie

cookies

传入 Cookie 字典。

resp.cookies

读取响应设置的 cookies。

10. Session 会话

requests.Session()

创建会话对象。它会自动保存 cookies，并复用连接。

适合场景：

  • 登录后继续请求
  • 多个请求共用 headers
  • 多个请求自动携带 cookies
  • 提高连续请求效率

12. 文件下载

小文件下载

大文件流式下载

使用 stream=True，避免一次性把文件全部读进内存。

resp.iter_content(chunk_size=1, decode_unicode=False)

分块读取响应体。

参数：

  • chunk_size：每块大小，单位 bytes
  • decode_unicode：是否解码成文本

13. 文件上传

files

上传文件。

files 字典的值可以是：

  • 文件对象
  • (filename, fileobj)
  • (filename, fileobj, content_type)
  • (filename, fileobj, content_type, headers)

14. SSL 证书

verify

是否验证 HTTPS 证书。

参数：

  • verify=True：默认，验证证书
  • verify=False：不验证证书，不推荐长期使用
  • verify="/path/to/cert.pem"：使用指定证书

15. 重定向

allow_redirects

是否允许自动重定向。

默认：

  • GET、OPTIONS、POST、PUT、PATCH、DELETE 会自动重定向
  • HEAD 默认不自动重定向

  • requests.exceptions.HTTPError：HTTP 状态码错误，由 raise_for_status() 抛出
  • requests.exceptions.RequestException：requests 异常基类

更正式的写法可以配合 urllib3.util.retry.Retry。

常用参数：

20. requests 和 urllib 的区别

  • requests 是第三方库，需要安装
  • urllib 是标准库，不需要安装
  • requests 更适合实际爬虫和 API 请求
  • urllib 更适合无依赖脚本或学习底层 URL/HTTP 处理
  • requests.Session() 处理登录态比 urllib 方便很多

21. 常见坑

  • 记得设置 timeout
  • resp.text 是字符串，resp.content 是 bytes
  • API 返回 JSON 时用 resp.json()，不要手动从 resp.text 乱切
  • 中文 URL 参数直接放进 params，不要自己拼接字符串
  • 登录后连续请求应该用 Session
  • 下载大文件要用 stream=True
  • verify=False 会降低 HTTPS 安全性，只适合临时调试
  • raise_for_status() 不会自动调用，需要你手动写
