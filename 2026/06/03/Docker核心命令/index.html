Docker 核心命令

分类： 编程基础
标签： Docker容器运维

日期：2026-06-03 20:00:00

Docker 的命令看起来很多，但真正常用的部分可以围绕四个对象来理解：镜像、容器、网络和数据卷。镜像负责提供运行环境，容器是镜像运行后的实例，网络负责容器之间以及容器与外部之间的通信，数据卷负责保存那些不应该随着容器删除而丢失的数据。

这篇文章整理 Docker 的核心命令、docker run 常见参数、数据卷挂载方式、inspect 查看细节的方法，以及 Docker Compose 的基本用法，适合作为日常使用时的速查笔记。

一、先理解 Docker 的几个核心对象

对象 含义 常见命令
镜像 Image 一个只读模板，包含应用运行所需的文件、依赖和环境 docker pull、docker images、docker image inspect
容器 Container 镜像运行后的实例，可以启动、停止、删除 docker run、docker start、docker stop、docker rm
网络 Network 控制容器之间、容器与宿主机之间如何通信 docker network create、docker network inspect
数据卷 Volume 独立于容器生命周期的数据存储空间 docker volume create、docker volume inspect

四、docker run 核心参数

docker run 是最常用也最容易写长的命令。它的参数主要用于配置容器运行方式、端口、挂载、环境变量和交互方式。

1. -d：后台运行

-d 是 detach，表示容器在后台运行，命令执行后会返回容器 ID，并释放当前终端。

如果不加 -d，容器会占用前台终端。对于 Nginx、MySQL、Redis 这类长期运行的服务，一般都会加 -d。

2. -p：端口映射

-p <host_port>:<container_port> 用于把宿主机端口映射到容器内部端口。

五、inspect：查看对象详细信息

inspect 是 Docker 中非常重要的排查命令。它不会修改对象，只负责查看详细元数据。

1. 查看容器信息

常见用途：

  • 查看容器 IP 地址
  • 查看挂载目录
  • 查看端口映射
  • 查看启动参数
  • 查看环境变量
  • 查看容器状态

2. 查看镜像信息

可以查看镜像默认命令、暴露端口、环境变量和层信息。

3. 查看数据卷信息

最常看的字段是 Mountpoint，它表示这个数据卷在宿主机上的真实位置。

4. 查看网络信息

可以查看网络驱动、子网、网关，以及连接到这个网络的容器。

2. host 网络模式

作用：让容器直接共享宿主机网络栈，不再拥有独立的网络隔离空间。

特点：

  • 网络性能开销较低
  • 容器端口会直接占用宿主机端口
  • 网络隔离性变弱

注意：--network host 更适合对网络性能要求很高、并且明确知道端口占用情况的场景。普通服务通常优先使用 -p 端口映射。

十二、总结

Docker 常用命令可以按对象来记：

  • 镜像：拉取、查看、检查
  • 容器：创建、运行、停止、删除
  • 网络：端口映射、自定义网络、host 模式
  • 数据卷：创建、挂载、检查、清理
  • Compose：把多个容器写进一个配置文件统一管理
