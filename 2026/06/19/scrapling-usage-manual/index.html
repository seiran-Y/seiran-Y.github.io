1. 项目简介与适用场景

功能定位

Scrapling 是一个 Python 网页抓取与解析库。它把 HTML 解析器、HTTP 客户端、Playwright 浏览器抓取、反检测浏览器会话和异步 Spider 框架放在同一套接口中。

常见使用场景：

  • 静态网页抓取：用 Fetcher 发送 HTTP 请求并直接得到可选择的 Response。
  • HTML 解析：用 Selector 对字符串 HTML 执行 CSS、XPath、文本、正则、相似元素查找。
  • 动态网页抓取：用 DynamicFetcher 打开 Chromium 等待页面渲染后提取 DOM。
  • 反检测抓取：用 StealthyFetcher 在更接近真实浏览器的环境中访问页面。
  • 中大型爬虫：用 Spider、CrawlSpider、SitemapSpider 管理请求队列、并发、去重、暂停恢复、结果导出。
  • 命令行提取：用 scrapling extract ... 把网页内容保存为 HTML、Markdown 或纯文本。
  • AI/MCP 工具调用：用 scrapling mcp 启动 MCP 服务，让支持 MCP 的客户端调用抓取能力。

常用参数

  • selector_config：传给最终 Selector / Response 的解析配置，例如 adaptive scraping 配置。
  • headers / extra_headers：HTTP fetcher 使用 headers，浏览器 fetcher 使用 extra_headers。
  • proxy / proxy_rotator：静态代理或代理轮换。
  • timeout：HTTP fetcher 单位是秒，浏览器 fetcher 单位是毫秒。
  • retries / retry_delay：失败重试次数和间隔。

返回对象/可继续调用的方法

所有 fetcher 返回的主要对象都是 Response。Response 继承 Selector，因此响应对象既有 HTTP 信息，也能直接 .css()、.xpath()、.find()。

适用场景

  • 想用少量代码完成“请求 + 解析”。
  • 需要在静态请求、浏览器请求、反检测请求之间切换。
  • 需要把单页抓取逐步升级为完整 Spider。

注意事项

  • 浏览器相关功能依赖 Playwright / Patchright 浏览器环境，安装后通常需要运行 scrapling install。
  • 不要用反检测功能绕过登录、付费墙、验证码或访问控制。
  • 生产环境应设置超时、重试、限速、代理失败处理和日志。

常用参数

Selector(content, url="", encoding="utf-8", huge_tree=True, adaptive=False, storage=SQLiteStorageSystem, keep_cdata=False, storage_args=None, keep_comments=False, adaptive_domain="")

  • content：HTML 字符串或字节。
  • url：当前页面 URL，用于 urljoin() 和相对链接处理。
  • encoding：文本编码。
  • huge_tree：允许解析更大的 HTML 树。
  • adaptive：启用自适应选择器定位。
  • storage / storage_args：adaptive scraping 的存储后端和参数。
  • keep_cdata / keep_comments：是否保留 CDATA 和注释。

选择器方法：

  • .css(selector, identifier="", adaptive=False, auto_save=False, percentage=40)
  • .xpath(selector, identifier="", adaptive=False, auto_save=False, percentage=40, **kwargs)
  • .find(*args, **kwargs)
  • .find_all(*args, **kwargs)
  • .find_by_text(text, first_match=True, partial=False, case_sensitive=False, clean_match=True)
  • .find_by_regex(query, first_match=True, case_sensitive=False, clean_match=True)
  • .find_similar(similarity_threshold=0.2, ignore_attributes=("href", "src"), match_text=False)
  • .re(regex, replace_entities=True, clean_match=False, case_sensitive=True)
  • .re_first(regex, default=None, replace_entities=True, clean_match=False, case_sensitive=True)

返回对象/可继续调用的方法

.css()、.xpath()、.find_all() 通常返回 Selectors，它是 list[Selector] 的增强版：

  • .get(default=None)：取第一个匹配的 HTML/text。
  • .getall()：取所有匹配结果。
  • .first / .last：取第一个或最后一个 Selector。
  • .length：匹配数量。
  • .filter(func) / .search(func)：按函数过滤或搜索。
  • .css() / .xpath() / .re() / .re_first()：继续链式提取。

单个 Selector 常用属性和方法：

  • .tag：标签名。
  • .text：当前元素清理后的文本。
  • .get_all_text(separator="\n", strip=False, ignore_tags=("script", "style"), valid_values=True)：提取完整文本。
  • .attrib：属性映射。
  • .html_content：内部 HTML。
  • .body：原始 body。
  • .prettify()：格式化 HTML。
  • .urljoin(relative_url)：拼接相对 URL。
  • .json()：把当前内容按 JSON 解析。

适用场景

  • 从已有 HTML 字符串中提取结构化数据。
  • 从 Fetcher / DynamicFetcher 返回的 Response 继续解析。
  • 页面结构可能小幅变化时，用 adaptive scraping 增加选择器韧性。

注意事项

  • ::text 和 ::attr(...) 是 CSS 选择器的提取扩展，适合直接取文本或属性。
  • .text 是元素文本；.get() 取决于当前选择器是否是文本/属性结果。
  • adaptive scraping 需要稳定的 identifier，否则无法复用保存过的定位信息。
  • 相似元素查找是启发式能力，不适合作为强一致业务逻辑的唯一依据。

常用参数

Fetcher.get(url, **kwargs)、Fetcher.post(url, **kwargs)、Fetcher.put(url, **kwargs)、Fetcher.delete(url, **kwargs)：

  • params：查询参数。
  • headers：请求头。
  • cookies：Cookie。
  • data：表单或原始请求体，常用于 POST/PUT。
  • json：JSON 请求体。
  • timeout：超时秒数，默认 30。
  • follow_redirects：是否跟随重定向，默认 "safe"，会拒绝跳转到内网/私有 IP。
  • max_redirects：最大重定向次数，默认 30，-1 表示不限。
  • retries：失败重试次数，默认 3。
  • retry_delay：重试间隔秒数，默认 1。
  • proxy：单个代理 URL，例如 "http://user:pass@host:8030"。
  • proxies：代理字典，例如 {"http": proxy_url, "https": proxy_url}。
  • proxy_auth：代理 Basic Auth，形如 ("user", "pass")。
  • auth：目标站点 Basic Auth。
  • verify：是否验证 TLS 证书。
  • cert：客户端证书。
  • impersonate：模拟浏览器，可传字符串或字符串列表；列表会随机选一个。
  • http3：启用 HTTP/3。
  • stealthy_headers：默认启用，自动添加更像真实浏览器的请求头。
  • selector_config：传递给最终 Response / Selector 的解析配置。

FetcherSession(...) 支持把上述大部分参数作为会话默认值。

返回对象/可继续调用的方法

返回 Response：

  • HTTP 信息：.status、.reason、.cookies、.headers、.request_headers、.history、.meta、.body。
  • 解析方法：.css()、.xpath()、.find()、.find_all()、.re()、.re_first()。
  • Spider 中可用：.follow() 创建新 Request。

适用场景

  • 静态 HTML 页面。
  • JSON/API 请求。
  • 需要 cookies、headers、代理、重试的请求脚本。
  • 大量 URL 并发抓取但不需要浏览器渲染。

注意事项

  • Fetcher 不执行 JavaScript，也不会等待前端接口完成。
  • http3=True 与 impersonate 同时使用时可能触发底层 curl 兼容问题。
  • follow_redirects="safe" 更安全，生产环境不建议无脑改成 True。
  • proxy_rotator 不能和 proxy / proxies 同时使用。

DynamicFetcher.fetch(url, **kwargs)、DynamicFetcher.async_fetch(url, **kwargs) 和 Dynamic Session 常用参数：

  • block_ads：阻止已知广告/追踪域名。
  • capture_xhr：匹配 XHR/fetch URL 的正则。
  • init_script：绝对路径 JS 初始化脚本。
  • user_data_dir：持久化用户数据目录。
  • locale、timezone_id、useragent：浏览器环境。
  • real_chrome：使用本机 Chrome channel。
  • cdp_url：连接已有浏览器 CDP WebSocket。
  • executable_path：指定浏览器可执行文件绝对路径。
  • dns_over_https：给 Chromium 添加 Cloudflare DoH 参数。

返回对象/可继续调用的方法

返回 Response，并额外适合检查：

  • .captured_xhr：捕获到的 XHR/fetch 响应列表。
  • .meta["proxy"]：如果使用代理轮换，可查看本次使用代理。
  • .css()、.xpath()：对渲染后的 DOM 提取内容。

Session 对象还支持：

  • .fetch(url, **kwargs)：同步或异步抓取。
  • .get_pool_stats()：查看页面池状态。
  • 上下文管理器自动关闭浏览器资源。

适用场景

  • 页面内容由 JavaScript 渲染。
  • 需要点击、滚动、登录态目录、设置 viewport。
  • 需要捕获前端接口返回。
  • 需要复用浏览器上下文提升性能。

注意事项

  • wait 和 timeout 是毫秒，不是秒。
  • init_script 和 executable_path 必须是存在的绝对文件路径。
  • proxy 和 proxy_rotator 不能同时使用。
  • disable_resources=True 可提速，但可能影响依赖图片、字体、XHR 的页面。
  • 使用 user_data_dir 时注意不要同时让多个进程写同一浏览器用户目录。

StealthyFetcher 支持大部分 DynamicFetcher 参数，并增加：

  • 页面检测自动化浏览器特征。

  • 需要处理 WebRTC、WebGL、Canvas 等指纹相关选项。

注意事项

  • 反检测功能不等于可以绕过授权、付费、登录或验证码。
  • solve_cloudflare=True 不保证一定成功，也可能需要更长等待时间。
  • 开启太多“隐藏”选项反而可能让浏览器特征异常，应按目标环境最小化配置。
  • 浏览器模式比 HTTP fetcher 慢，优先只对必要页面使用。

返回对象/可继续调用的方法

  • ProxyRotator.get_proxy()：获取下一个代理。

适用场景

  • 需要通过固定代理访问目标站点。
  • 代理池中个别代理可能失败，需要自动轮换。
  • 浏览器页面加载太慢，需要阻断图片、字体、广告和追踪脚本。
  • 代理场景下希望降低 DNS 泄漏风险。

注意事项

  • 不要同时设置 proxy 和 proxy_rotator。
  • 浏览器代理轮换通常会为每个代理创建新的 context，资源消耗更高。
  • disable_resources=True 可能导致页面布局或脚本行为变化。
  • 代理质量、地域、协议和认证格式会直接影响成功率。

  • concurrent_requests：全局并发数，默认 4。
  • concurrent_requests_per_domain：单域名并发，默认 0 表示不单独限制。
  • download_delay：请求间隔秒数。
  • max_blocked_retries：被判定 blocked 后最大重试次数。
  • fp_include_kwargs、fp_keep_fragments、fp_include_headers：请求去重指纹参数。
  • logging_level、logging_format、logging_date_format、log_file：日志配置。

Spider 实例化：

  • Spider(crawldir=None, interval=300.0)：传入 crawldir 后启用 checkpoint，interval 为定期保存秒数。

Request(url, sid="", callback=None, priority=0, dont_filter=False, meta=None, **kwargs)：

  • sid：选择哪个 session。
  • callback：响应回调，默认通常是 parse。
  • priority：优先级，数值越高越优先。
  • dont_filter：是否绕过去重。
  • meta：请求元数据，会传到响应。
  • **kwargs：传给对应 session 的抓取参数，例如 headers、extra_headers、method、json、wait_selector。

返回对象/可继续调用的方法

spider.start() 返回 CrawlResult：

  • .stats：CrawlStats。
  • .items：ItemList。
  • .paused：是否暂停。
  • .completed：是否正常完成。
  • len(result)：抓取 item 数。
  • for item in result：遍历 item。

ItemList：

  • .to_json(path, indent=False)：导出 JSON。
  • .to_jsonl(path)：导出 JSON Lines。

CrawlStats：

  • .to_dict()：导出统计字典。
  • .elapsed_seconds、.requests_per_second：运行指标。
  • 常见字段：requests_count、failed_requests_count、items_scraped、items_dropped、blocked_requests_count、response_status_count、sessions_requests_count。

Response.follow(...)：

  • 基于当前响应创建新 Request。
  • 支持相对 URL 自动补全。
  • 默认继承前一个请求的 session 参数，并自动设置 referer。

适用场景

  • 多页分页抓取。
  • 需要去重、限速、并发和结果导出。
  • 需要跨 HTTP 和浏览器 session 混合抓取。
  • 需要长任务暂停/恢复。

注意事项

  • Spider.name 必须设置，否则初始化会报错。
  • 如果没有 start_urls，必须重写 start_requests()。
  • parse() 必须是异步生成器，使用 yield 产出 item 或 Request。
  • 如果重写 configure_sessions()，至少要 manager.add(...) 一个 session。
  • stream() 模式不支持 SIGINT pause/resume。

常用参数

排障时优先检查：

  • status：是否 200、301/302、403、429、500。
  • body：目标内容是否在 HTML 中。
  • headers / request_headers：请求头是否符合预期。
  • timeout：页面是否因等待不足失败。
  • wait_selector：动态页面是否等待了正确元素。
  • network_idle：前端接口是否仍在加载。
  • retries / retry_delay：是否有短暂网络波动。
  • proxy / proxy_rotator：代理是否可用。

返回对象/可继续调用的方法

问题定位通常从 Response 开始：

  • .status 判断请求结果。
  • .body 保存原始响应。
  • .css() / .xpath() 验证选择器。
  • .captured_xhr 查看浏览器捕获接口。
  • .meta 查看代理等元数据。

Spider 中从 CrawlResult 开始：

  • result.stats.to_dict() 查看失败、被 robots 拦截、blocked、状态码分布。
  • result.items.to_json(...) 导出样本检查。
  • log_file 保存日志。

适用场景

  • 选择器无结果。
  • HTTP 请求可访问但内容为空。
  • 浏览器页面没有等到目标元素。
  • 代理失败或频繁 403/429。
  • Spider 结果数量和预期不一致。

注意事项

  • 如果静态 HTML 里没有目标内容，通常需要 DynamicFetcher。
  • 如果 DynamicFetcher 仍抓不到，检查是否需要登录、滚动、点击、等待 XHR。
  • 403/429 不一定是“反检测没做好”，也可能是访问频率、地域、权限或服务条款限制。
  • 遵守 robots.txt、版权、隐私和目标网站服务条款。
  • 设置合理并发和延迟，不要对目标服务造成压力。
  • 不要抓取或保存敏感个人信息，除非你有明确合法授权和安全存储方案。
